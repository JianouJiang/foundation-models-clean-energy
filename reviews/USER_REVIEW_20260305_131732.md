## PART A: Bibliometric & Scientometric Analysis

Build a full Python pipeline in `codes/` that programmatically analyzes the research landscape.

### Data Collection (`codes/data_processing/collect_papers.py`)
- Use the OpenAlex API (https://api.openalex.org — free, no key needed) to collect metadata for 500-2000+ papers
- Queries: ("foundation model" OR "large language model" OR "LLM" OR "GPT" OR "vision language model" OR "multimodal") AND ("energy" OR "solar" OR "wind" OR "hydropower" OR "renewable" OR "turbine" OR "photovoltaic" OR "grid" OR "power system")
- Date range: 2017-2026
- Collect: title, abstract, year, venue, citation count, authors, institutions, countries, keywords, open access status, referenced works
- Save raw: `data/openalex_papers.json`

### Automated Classification (`codes/data_processing/classify_papers.py`)
- Build a keyword-based + TF-IDF classifier to auto-tag every paper by:
  - Energy domain (hydro / wind / solar / grid / storage / ecological / multi)
  - FM type (LLM / VLM / time-series FM / diffusion / multimodal / standard transformer)
  - Task (forecasting / fault detection / anomaly / NLP / image / optimization / control / other)
  - Deployment strategy (zero-shot / few-shot / fine-tuned / RAG / agent / not specified)
- Save: `data/classified_papers.csv`

### Scientometric Analysis (`codes/analysis/scientometrics.py`)
- Publication growth by year with exponential/logistic fit + R² and doubling time
- Citation network analysis — identify the most influential papers and citation clusters
- Author collaboration network — who is working with whom across countries
- Keyword co-occurrence network with community detection (Louvain clustering)
- Journal impact mapping — which venues publish the most FM+energy work
- Geographic hotspot analysis — which countries and institutions lead
- Temporal keyword evolution — how the vocabulary shifted from "deep learning" to "foundation model" to "LLM agent"
- Compute a "research maturity index" for each (FM type × energy domain) cell

### Figures from Bibliometrics
- Publication trend with exponential fit (by FM type)
- Stacked area chart by energy domain
- **Taxonomy heatmap**: FM capability × energy task — cell intensity = paper count, annotated with specific counts. The GAPS are the contribution.
- Keyword co-occurrence network (colored by community)
- Geographic bar chart or map
- PRISMA flow diagram
- Temporal keyword evolution (alluvial/sankey diagram or streamgraph)

## PART B: Benchmark Experiments — Actually Run Foundation Models on Energy Data

This is what separates a real research paper from a summary. Run real experiments.

### Experiment 1: Time-Series Foundation Models for Energy Forecasting
(`codes/models/timeseries_benchmark.py`)
- Download a public energy time-series dataset (e.g., UCI Household Power Consumption, or ENTSO-E electricity load data, or a public wind/solar dataset from Kaggle/Zenodo)
- Compare:
  - Classical baselines: ARIMA, XGBoost
  - Standard DL: LSTM, vanilla Transformer
  - Foundation time-series models: use HuggingFace implementations of Lag-Llama, MOMENT, or Chronos (Amazon) — these are open-weight and pip-installable
- Metrics: MAE, RMSE, MAPE on held-out test set
- Key result: Do foundation time-series models actually outperform classical methods on energy data? The answer (yes/no/it depends) is genuinely useful.
- Zero-shot vs fine-tuned comparison if feasible

### Experiment 2: LLM for Energy Domain Question Answering
(`codes/models/llm_energy_qa.py`)
- Curate a small energy domain Q&A benchmark: 50-100 questions about hydropower operations, wind turbine maintenance, solar panel specs, grid stability concepts
- Source questions from textbooks, technical manuals, or generate from domain knowledge
- Test via API or local model:
  - GPT-4 / Claude (via API if keys available)
  - Open-weight: Qwen2.5 (already installed locally via Ollama!)
  - Domain-specific if available
- Score: accuracy, hallucination rate, domain-specificity of answers
- Key result: How well do general-purpose LLMs handle energy domain knowledge without fine-tuning?

### Experiment 3: Vision-Language Model for Energy Infrastructure Inspection
(`codes/models/vlm_inspection.py`)
- Find a public solar panel defect dataset (e.g., ELPV dataset on GitHub — electroluminescence images of solar cells with defect labels) or wind turbine blade damage images
- Test zero-shot classification with CLIP (pip install open-clip-torch):
  - Define text prompts: "a photo of a defective solar panel", "a photo of a healthy solar panel"
  - Compute CLIP similarity scores and classify
- Compare against a simple fine-tuned ResNet baseline
- Key result: Can VLMs do zero-shot energy infrastructure inspection? How does zero-shot compare to supervised?

### Experiment 4: RAG for Energy Technical Documents
(`codes/models/rag_pipeline.py`)
- Build a simple RAG pipeline:
  - Collect 10-20 public energy technical documents (e.g., IEA reports, IRENA publications — PDFs freely available)
  - Chunk documents, embed with a sentence transformer (all-MiniLM-L6-v2 or similar, pip-installable)
  - Store in a simple FAISS index
  - Query with energy-specific questions
  - Generate answers using Ollama Qwen2.5 (local, free)
- Compare: RAG-augmented answers vs. direct LLM answers (no retrieval)
- Metrics: answer relevance, factual grounding, hallucination reduction
- Key result: Does RAG improve LLM accuracy on energy domain questions?

### Experiment 5: Reproducibility Audit of Reviewed Papers
(`codes/analysis/reproducibility_audit.py`)
- For the top 100 most-cited papers in the collected database:
  - Check if they have a GitHub link (search OpenAlex, Semantic Scholar, or Papers With Code API)
  - Check if code is available, if data is available, if results are reproducible
- Compute: % with open code, % with open data, % with both
- This directly supports AIDER's mission of open-process research
- Key result: How reproducible is the FM+energy literature? (Probably very low — that's a finding)
